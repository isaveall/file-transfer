# 📁 局域网文件传输工具 (File Transfer)

Win11 风格的局域网文件传输应用，支持拖拽发送、自动发现设备。

## 界面预览

```
┌─────────────────────────────────────────────────────┐
│  发送到: [Mac Mini (192.168.1.100) ▼]  🔄  本机: PC │
├─────────────────────────────────────────────────────┤
│  文件              │ 大小    │ 进度   │ 状态        │
├─────────────────────────────────────────────────────┤
│                     ┌──────────────┐                │
│                     │              │                │
│                     │  ⬇️ 拖拽到此  │                │
│                     │              │                │
│                     └──────────────┘                │
├─────────────────────────────────────────────────────┤
│  [+]  [-]  [-]                 [取消]    [发送]     │
└─────────────────────────────────────────────────────┘
```

## 功能特性

- ✅ **自动发现局域网设备**（UDP 广播）
- ✅ **拖拽文件/文件夹** 到窗口发送
- ✅ **选择文件/文件夹** 按钮添加
- ✅ **实时进度显示**（单文件 + 总进度）
- ✅ **接收文件自动保存** 到 `~/Downloads/FileTransfer/`
- ✅ **无边框窗口**，Win11 深色主题风格
- ✅ **手动输入 IP** 支持跨网段传输

## 安装 & 运行

### 前置要求
- **Node.js** v18+
- **npm** 或 **yarn**

### 步骤

```bash
# 1. 进入项目目录
cd file-transfer-app

# 2. 安装依赖（含 Electron）
npm install

# 3. 启动应用
npm start
```

### 打包为 .exe（可选）

```bash
# 使用 electron-builder 打包
npm install --save-dev electron-builder
npx electron-builder --win
```

## 使用方法

### 发送方
1. 启动应用，自动扫描局域网设备
2. 从下拉菜单选择目标设备
3. 拖拽文件/文件夹到窗口，或点击 `+` 按钮添加
4. 点击「发送」开始传输

### 接收方
1. 只需在同一局域网内启动应用
2. 自动开启 TCP 监听（端口 34567）
3. 收到文件后自动保存到 `Downloads/FileTransfer/`
4. 底部显示接收进度

## 技术架构

```
┌─────────────────────────────────┐
│  Electron (主进程)              │
│  ┌───────────┐ ┌─────────────┐ │
│  │ TCP Server│ │ UDP Socket  │ │
│  │ (接收文件) │ │ (设备发现)  │ │
│  └───────────┘ └─────────────┘ │
└──────────────┬──────────────────┘
               │ IPC
┌──────────────▼──────────────────┐
│  Renderer (UI)                  │
│  HTML + CSS + JS                │
│  拖拽文件 · 进度显示 · 设备列表 │
└─────────────────────────────────┘
```

### 通信协议
- **设备发现**: UDP 广播 (端口 34568)
- **文件传输**: TCP (端口 34567)
- **头部**: 文件数量 + (文件名长度 + 文件名 + 文件大小) × N
- **数据**: 文件原始字节流

## 注意事项

- 确保防火墙允许 `34567/TCP` 和 `34568/UDP`
- 同一局域网内的设备才能自动发现
- 跨网段设备需使用「手动输入 IP」
- 大文件传输建议使用有线网络
